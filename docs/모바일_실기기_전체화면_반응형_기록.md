# 모바일 실기기 전체화면 반응형 기록

## 작업 목적

고객용 앱을 휴대폰 Chrome에서 열었을 때 화면 안에 다시 아이폰 목업 프레임이 표시되어, 실제 앱처럼 보이지 않는 문제가 있었다.

데스크톱에서는 발표용 목업 프레임을 유지하고, 모바일에서는 실제 휴대폰 앱처럼 전체 화면으로 보이도록 반응형 처리를 적용했다.

## 적용 내용

- 모바일 화면에서는 검은 아이폰 외곽 프레임을 제거했다.
- 모바일 화면에서는 목업용 다이나믹 아일랜드를 숨겼다.
- 모바일 화면에서는 목업용 상태바를 숨겼다.
- 모바일 화면에서는 앱 컨테이너가 `100dvh` 높이로 화면을 채우도록 했다.
- 데스크톱 화면에서는 기존 아이폰 목업 프레임을 유지했다.

## 수정 파일

```text
src/app/page.tsx
```

## 검증

```bash
npm.cmd run build
```

빌드 성공을 확인했다.

## 로컬 휴대폰 확인 메모

로컬 휴대폰 테스트 시 카메라 권한은 `https` 환경에서만 정상 동작한다.

```text
https://{PC_IP}:3000
```

로컬 자체 인증서 경고가 뜨면 `고급` 또는 `세부정보`를 열고 `안전하지 않음으로 이동`을 선택해야 한다.

실제 배포 환경에서는 Cloudflare HTTPS가 적용된 아래 주소에서 확인하는 것이 가장 안정적이다.

```text
https://swapit.my
```
