# 로그인 후 예약 정보 복구 구현 기록

## 작업 배경

고객용 앱에서 데모 로그인 후 예약을 진행해도, 로그아웃 후 다시 로그인하면 홈 화면에서 기존 예약 정보가 보이지 않는 문제가 있었다.

## 구현 내용

1. 로그인 성공 후 `GET /api/swap-requests/latest?userId={userId}` API를 호출하도록 연결했다.
2. 최신 신청에 예약/바로콜 정보가 있으면 홈 화면 예약 카드에 다시 표시하도록 했다.
3. 브라우저에 저장된 데모 사용자 정보가 있을 때도 앱 진입 시 최신 신청 정보를 자동 복구하도록 했다.
4. 최신 신청이 없거나 조회에 실패하면 새 신청을 시작할 수 있도록 빈 상태로 정리한다.

## 기대 동작

- 같은 계정으로 다시 로그인하면 기존 예약 정보가 다시 표시된다.
- 다른 계정으로 로그인하면 해당 계정의 최신 신청 정보만 표시된다.
- 예약 정보가 있는 경우 홈 화면에서 예약 시간과 주소를 다시 확인할 수 있다.

## 확인 사항

- 프론트 빌드: `npm.cmd run build` 성공

## 대화 요약

사용자는 “다시 로그인하면 예약 정보도 기억하고 있어야 하지 않냐”고 확인했다. 이에 따라 로그인 직후 사용자 ID 기반으로 최신 교환 신청을 조회하고, 프론트 상태를 DB 기준으로 다시 채우는 방식으로 수정했다.
