# 고객용 앱 휴대폰 Chrome 접속 가이드

## 1. 이 문서는 언제 쓰나요?
팀원이 자기 휴대폰 Chrome에서 고객용 앱을 직접 열어보고 싶을 때 사용합니다.

고객용 앱은 PC에서 개발 서버를 켜고, 휴대폰에서 같은 와이파이로 접속하는 방식입니다.

---

## 2. 준비물

- 고객용 앱 코드가 있는 PC
- 휴대폰
- PC와 휴대폰이 같은 Wi-Fi에 연결되어 있어야 함
- PC에서 고객용 프론트 서버가 켜져 있어야 함
- 백엔드 기능까지 확인하려면 백엔드 서버도 켜져 있어야 함

---

## 3. 가장 중요한 개념

휴대폰에서 아래 주소로 접속하면 안 됩니다.

```text
http://localhost:3000
```

이유는 `localhost`가 휴대폰에서는 PC가 아니라 휴대폰 자기 자신을 뜻하기 때문입니다.

휴대폰에서는 PC의 Wi-Fi IP 주소로 접속해야 합니다.

예시:

```text
http://172.30.1.55:3000
```

---

## 4. PC의 Wi-Fi IP 주소 확인하기

PC에서 PowerShell을 열고 아래 명령어를 입력합니다.

```powershell
ipconfig
```

출력 중에서 아래와 비슷한 항목을 찾습니다.

```text
무선 LAN 어댑터 Wi-Fi
IPv4 주소 . . . . . . . . . : 172.30.1.55
```

여기서 `172.30.1.55` 같은 값이 PC의 Wi-Fi IP 주소입니다.

팀원마다 숫자는 다를 수 있습니다.

---

## 5. 백엔드 서버 켜기

백엔드까지 연결해서 보려면 먼저 백엔드 서버를 켭니다.

PowerShell에서 아래 폴더로 이동합니다.

```powershell
cd C:\Users\yeosa\Desktop\DX\beyond404-server
```

서버 실행:

```powershell
.\gradlew.bat bootRun
```

정상 실행되면 백엔드는 보통 아래 주소로 열립니다.

```text
http://localhost:8080
```

휴대폰 기준 백엔드 주소는 아래처럼 PC IP를 사용합니다.

```text
http://PC_IP주소:8080
```

예시:

```text
http://172.30.1.55:8080
```

---

## 6. 고객용 프론트 서버 켜기

새 PowerShell 창을 열고 고객용 앱 폴더로 이동합니다.

```powershell
cd C:\Users\yeosa\Desktop\DX\beyond404-web
```

처음 실행하는 PC라면 패키지를 설치합니다.

```powershell
npm install
```

그 다음 프론트 서버를 외부 접속 가능 상태로 실행합니다.

```powershell
npm run dev -- -H 0.0.0.0 -p 3000
```

정상 실행되면 PC에서는 아래 주소로 볼 수 있습니다.

```text
http://localhost:3000
```

휴대폰에서는 아래처럼 접속합니다.

```text
http://PC_IP주소:3000
```

예시:

```text
http://172.30.1.55:3000
```

---

## 7. 휴대폰 Chrome에서 접속하기

휴대폰 Chrome 주소창에 아래 형식으로 입력합니다.

```text
http://PC_IP주소:3000
```

예시:

```text
http://172.30.1.55:3000
```

접속이 되면 고객용 앱 화면이 휴대폰에 표시됩니다.

---

## 8. 접속이 안 될 때 확인할 것

### 8.1 PC와 휴대폰이 같은 Wi-Fi인지 확인

PC는 집 Wi-Fi, 휴대폰은 데이터/LTE/5G면 접속이 안 됩니다.

휴대폰도 반드시 같은 Wi-Fi에 연결해야 합니다.

### 8.2 주소에 localhost를 쓰지 않았는지 확인

휴대폰에서는 아래 주소가 아닙니다.

```text
http://localhost:3000
```

반드시 PC IP를 사용해야 합니다.

```text
http://PC_IP주소:3000
```

### 8.3 프론트 서버를 0.0.0.0으로 실행했는지 확인

아래처럼 실행해야 휴대폰에서 접근할 수 있습니다.

```powershell
npm run dev -- -H 0.0.0.0 -p 3000
```

그냥 `npm run dev`만 실행하면 PC에서만 열리고 휴대폰에서는 안 될 수 있습니다.

### 8.4 Windows 방화벽 허용 확인

처음 서버를 켤 때 Windows 방화벽 팝업이 뜨면 `허용`을 눌러야 합니다.

팝업을 닫았거나 차단했다면 휴대폰 접속이 막힐 수 있습니다.

### 8.5 백엔드 연결 오류가 뜰 때

앱 화면은 열리는데 `연결이 원활하지 않습니다` 같은 메시지가 뜨면 백엔드 주소 문제일 수 있습니다.

프론트에서 API 주소가 `localhost:8080`으로 되어 있으면, 휴대폰에서는 백엔드를 못 찾습니다.

휴대폰 테스트 시 API 주소는 아래처럼 PC IP를 바라봐야 합니다.

```text
http://PC_IP주소:8080
```

예시:

```text
http://172.30.1.55:8080
```

---

## 9. 카메라 기능 주의사항

휴대폰 Chrome에서 카메라 기능을 쓰려면 브라우저 보안 정책 때문에 제한이 있을 수 있습니다.

일반적으로 카메라는 아래 환경에서 잘 동작합니다.

- `https://` 주소
- `localhost`
- 배포된 HTTPS 사이트

같은 Wi-Fi의 `http://PC_IP주소:3000` 주소에서는 기기나 브라우저 설정에 따라 카메라 권한이 막힐 수 있습니다.

카메라까지 안정적으로 테스트하려면 나중에 아래 방식 중 하나가 좋습니다.

- HTTPS 배포 주소 사용
- ngrok 같은 HTTPS 터널 사용
- AWS Amplify 배포 주소 사용

---

## 10. 팀원에게 가장 짧게 안내하는 버전

1. PC와 휴대폰을 같은 Wi-Fi에 연결한다.
2. PC에서 `ipconfig`로 IPv4 주소를 확인한다.
3. 백엔드 실행: `beyond404-server`에서 `./gradlew.bat bootRun`
4. 프론트 실행: `beyond404-web`에서 `npm run dev -- -H 0.0.0.0 -p 3000`
5. 휴대폰 Chrome에서 `http://PC_IP주소:3000`으로 접속한다.
6. 안 되면 방화벽, Wi-Fi, IP 주소, 서버 실행 여부를 확인한다.
