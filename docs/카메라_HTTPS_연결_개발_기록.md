# 카메라 HTTPS 연결 개발 기록

## 작업 일시
- 2026-06-11

## 작업 배경
- 고객용 앱의 STEP 1 촬영 화면에서 실제 카메라 권한 팝업이 뜨지 않고, 데모 촬영 안내만 표시되는 문제가 있었다.
- 사용자는 가전 외관과 모델 라벨을 실제 카메라로 촬영한 뒤 Mock VLM 인식 단계로 넘어가는 흐름을 원했다.
- 모바일/브라우저에서 카메라 API는 보안 주소에서만 동작하므로, 로컬 확인을 위해 HTTPS 개발 서버 구성이 필요했다.

## 대화에서 확인한 문제
- `http://localhost:3000` 접속 시 HTTPS 서버와 프로토콜이 맞지 않아 `ERR_EMPTY_RESPONSE`가 발생했다.
- `https://localhost:3000` 접속 시 자체 서명 인증서 경고가 표시되지만, `고급 → 계속 진행`으로 로컬 앱 진입이 가능하다.
- `http://PC_IP:3000` 또는 보안이 아닌 주소에서는 브라우저가 `navigator.mediaDevices.getUserMedia`를 막아 카메라 권한 팝업 자체가 뜨지 않을 수 있다.

## 구현 내용
- 촬영 시작 버튼 클릭 시 카메라 권한 요청이 직접 실행되도록 수정했다.
- 카메라 스트림을 `video` 태그에 연결하고, 촬영 결과를 실제 비디오 해상도 기준으로 캡처하도록 수정했다.
- 촬영 단계에서는 휴대폰 프레임 상단의 흰 상태바 영역이 보이지 않도록 전체 화면 카메라 레이아웃을 조정했다.
- 프론트 API 호출 기본값을 브라우저에서는 같은 origin의 `/api`로 보내도록 수정했다.
- Next.js rewrite를 통해 `/api/*` 요청이 백엔드 `http://127.0.0.1:8080`으로 전달되게 유지했다.
- 로컬 HTTPS 개발 실행용 스크립트 `dev:https`를 추가했다.
- 로컬 개발 인증서 폴더 `certificates/`가 Git에 올라가지 않도록 `.gitignore`에 추가했다.

## 확인 방법
1. 백엔드 서버를 실행한다.
2. 고객용 앱을 HTTPS 개발 서버로 실행한다.
   ```powershell
   npm run dev:https -- -H 0.0.0.0 -p 3000
   ```
3. 브라우저에서 아래 주소로 접속한다.
   ```text
   https://localhost:3000
   ```
4. 인증서 경고가 뜨면 `고급 → 계속 진행`을 선택한다.
5. 앱에서 촬영 화면으로 이동한 뒤 `동의 후 촬영 시작`을 누른다.
6. 카메라 권한 팝업이 뜨면 허용한다.

## 검증 결과
- `npm run build` 성공.
- `https://localhost:3000` 응답 200 확인.
- `https://localhost:3000/api/auth/demo-login` 요청이 백엔드로 프록시되어 정상 응답하는 것 확인.

## 남은 주의사항
- 휴대폰에서 `https://PC_IP:3000`으로 접속할 경우 자체 서명 인증서를 모바일 브라우저가 강하게 막을 수 있다.
- 실제 휴대폰 카메라 데모를 안정적으로 보여주려면 Cloudflare Tunnel, 정식 도메인 HTTPS, 또는 AWS 배포 환경을 사용하는 것이 좋다.
