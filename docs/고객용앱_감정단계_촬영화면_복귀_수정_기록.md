# 고객용앱 감정단계 촬영화면 복귀 수정 기록

## 작업 배경
- 고객용 앱에서 사진 촬영 후 `감정가 확인하기` 버튼을 누르면 감정 화면으로 진행되지 않고 촬영 전 동의/촬영 단계로 돌아가는 문제가 있었다.
- 화면 상단에 `일시적인 오류가 발생했습니다` 안내가 남아 사용자가 어디서 다시 진행해야 하는지 알기 어려웠다.

## 원인
- 감정 API 오류가 전체 SwapIt 화면 공통 오류로 처리되어 동의/촬영 화면에도 계속 표시되었다.
- 감정 오류와 다른 API 오류가 분리되지 않아 감정 단계의 재시도 흐름이 명확하지 않았다.

## 수정 내용
- 감정 API 오류를 전체 공통 오류에서 분리했다.
- 감정 실패 시 촬영 전 단계로 튕기지 않고 감정 화면 안에서 재시도할 수 있도록 변경했다.
- 감정 화면의 한글 깨짐 문구를 정상 한국어로 복구했다.
- `다시 감정하기`, `다시 촬영하기` 액션을 명확히 분리했다.
- 사진 분석 요청에 `imageUrl` 값을 함께 전달해 백엔드 mock 분석 요청 데이터가 더 안정적으로 들어가도록 보완했다.
- 배포 브라우저에서는 `NEXT_PUBLIC_API_BASE_URL` 대신 같은 도메인의 `/api` 경로를 사용하도록 수정했다.
  - 이유: 배포 빌드에 `127.0.0.1:8080` 같은 로컬 백엔드 주소가 포함되면, 사용자 브라우저가 사용자 기기의 8080 포트를 찾게 되어 감정 API 호출이 실패한다.
  - 서버 사이드 실행에서는 기존처럼 `NEXT_PUBLIC_API_BASE_URL` 또는 `http://127.0.0.1:8080`을 사용한다.

## 확인 결과
- `npm run build` 정상 통과.
- 로컬 고객용 앱 `http://localhost:3000` 응답 확인.

## 대화 메모
- 사용자는 배포 전 로컬에서 먼저 확인한 뒤 GitHub와 배포 서버에 반영하고 싶다고 했다.
- 로컬 서버가 `ERR_EMPTY_RESPONSE`를 보여 고객용 앱 dev 서버를 재시작했고, 이후 정상 접속을 확인했다.
